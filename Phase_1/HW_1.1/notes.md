第一周学习笔记：环境筑基与踩坑实录
一、 虚拟机与 Ubuntu 安装心得
这次环境搭建可以说是“万事开头难”的真实写照。从一开始去 Broadcom 官网下载 VMware Workstation Pro 就遭遇了找不到入口、账号合规验证、下载按钮禁止点击等一系列复杂的流程。
在安装 Ubuntu 22.04 LTS 时，初步体会到了给系统分配虚拟资源的逻辑。最深刻的教训是：Linux 系统和 Windows 是完全两个不同的生态。习惯了在 Windows 下双击运行 .exe 文件，结果在 Ubuntu 里下载了 Windows 版的安装包直接报错打不开，这才学到 Ubuntu 下专属的软件包后缀通常是 .deb，并且需要使用终端命令来正确安装。
二、 Linux 常用命令与快捷键总结
脱离图形界面进入纯命令行终端操作，一开始非常不适应，但掌握了以下几个核心指令和快捷键后，逐渐找到了感觉：
快捷键 Ctrl + Alt + T：快速呼出终端黑框。
快捷键 Ctrl + Shift + V：在终端中粘贴文本（习惯性按 Ctrl+V 是无效的）。
cd <目录路径>：切换当前所在的工作目录（例如 cd ~/Downloads）。
sudo apt update：更新本地的软件源索引。
wget <链接地址>：直接在命令行里从网络下载文件。
bash <脚本名.sh>：运行 Shell 脚本文件（安装 Miniconda 时用到）。
sudo apt install ./<包名.deb>：安装本地的 .deb 软件包（实操技巧：输入文件名前几个字母后按 Tab 键可以自动补全超长的文件名）。
sed 替换命令：用于在终端里直接修改文件内容，这次用它把系统默认源替换成了清华镜像源（实操中还踩了多写一个 cn. 导致域名无法解析的坑，也是用它修正的）。
三、 Conda 隔离原理的个人理解
为什么需要隔离？
在接触 Conda 之前，以为 Python 装一次就可以随便用。但实际工程中，不同的算法项目往往依赖同一个库（比如 OpenCV）的完全不同的版本。如果把所有包都装在同一个大环境里，不同版本的包互相覆盖、冲突，会导致项目直接崩溃。
隔离的本质：
Conda 就像是给每个项目单独建了一个虚拟环境。例如这次创建的 robocon_env 环境，它就像一个独立的集装箱，里面装了专属的 Python 3.10 解释器，以及专供这个项目使用的 Numpy 和 OpenCV 库。我们在里面怎么折腾，都不会弄坏外面的 (base) 基础环境。
实操感悟：
每次需要工作时，必须通过 conda activate robocon_env 进门，终端最左侧的括号前缀会变成当前环境名，明确标示当前位置。
开源工具的规则也在实时变动，例如这次就遇到 Conda 官方必须先通过 conda tos accept 命令手动同意服务条款后，才能正常创建环境和下载包。
