# 1.2 阶段进度笔记：Python 基础与脚本编写复盘

在编写批量图片处理脚本 `img_checker.py` 时，我遇到并解决了以下 Python 语法与工程问题：

## 1. 字典（Dict）的遍历方式
一开始对如何打印字典中的嵌套数据有些生疏。
**解决方法：** 使用了 `dict.items()` 方法。它可以同时解包字典的键（图片名称）和值（包含分辨率和通道数的子字典），通过 `for name, info in img_info_dict.items():` 的方式，配合 f-string 格式化输出，代码既简洁又清晰。

## 2. 浅拷贝与深拷贝的区别（拓展思考）
虽然本次脚本中的字典结构相对简单，每次循环都是重新赋值，但在将变量存入字典时联想到了拷贝问题。
如果直接把一个可变对象（如列表或另一个复杂字典）赋值给字典的值，它其实是“浅拷贝”（传引用）。如果后续修改了原对象，字典里的值也会跟着变。为了避免这种隐患，在使用复杂数据结构时，应该引入 `copy` 模块，使用 `copy.deepcopy()` 进行深拷贝，开辟独立的内存空间。

## 3. OpenCV 图像形状的维度陷阱
在提取分辨率时发现一个反直觉的设定：
传统我们说分辨率是“宽 x 高”，但 OpenCV 的 `img.shape` 返回的元组顺序是 `(height, width, channels)`。因此在提取变量时，必须写成 `height, width, channels = img.shape`，然后在打印时手动调换位置变成 `width x height`，防止长宽比颠倒。
